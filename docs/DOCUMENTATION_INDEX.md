## üìñ Documentation Best Practices

### Writing Guidelines
1. **Clear and Concise**: Use simple language
2. **Examples**: Include code examples
3. **Diagrams**: Use ASCII art or mermaid
4. **Links**: Cross-reference related docs
5. **Updates**: Keep version and date

### Structure
```markdown
# Title

> Brief description

## Table of Contents
1. [Section](#section)

## Section
Content here...

### Subsection
More details...

---

**Last Updated**: YYYY-MM-DD
```

### Code Examples
```typescript
// Always include:
// 1. Imports
// 2. Types/interfaces
// 3. Complete, working code
// 4. Comments explaining key parts

import { example } from '@/lib/example';

interface ExampleProps {
  value: string;
}

export function Example({ value }: ExampleProps) {
  // Implementation
  return <div>{value}</div>;
}
```

## ü§ù Contributing to Documentation

### How to Contribute

1. **Find what needs updating**
   - Check "In Progress" or "Planned" sections
   - Look for outdated information
   - Identify gaps

2. **Make changes**
   - Follow writing guidelines
   - Include examples
   - Add diagrams if helpful

3. **Submit PR**
   - Clear description of changes
   - Tag relevant reviewers
   - Link to related issues

### Review Process

1. Technical accuracy check
2. Grammar and style review
3. Link verification
4. Example validation
5. Approval and merge
